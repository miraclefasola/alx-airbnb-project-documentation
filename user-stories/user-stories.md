The following **user stories** were derived from the use case diagram.  
They define the main system interactions across **user authentication**, **property management**, **booking**, **payments**, and **reviews**.

---

## 1. User Authentication & Authorization

**Story 1:**  
As a **user**, I want to be able to **register an account** so that I can **access the platform and use available features**.

**Story 2:**  
As a **registered user**, I want to **log in securely** so that I can **access my personalized dashboard**.


---

## 2. Property Management

**Story 4:**  
As a **host**, I want to **create, update, and delete property listings** so that I can **manage my available rentals easily**.

**Story 5:**  
As a **guest**, I want to **browse and filter properties** by price, location, and amenities so that I can **find suitable options quickly**.

**Story 6:**  
As an **admin**, I want to **verify or remove property listings** that violate policies so that I can **maintain platform integrity**.

---

## 3. Booking Management

**Story 7:**  
As a **guest**, I want to **book a property for specific dates** so that I can **secure my stay in advance**.

**Story 8:**  
As a **host**, I want to **approve or decline booking requests** so that I can **manage my propertyâ€™s availability effectively**.

**Story 9:**  
As a **guest**, I want to **cancel a booking under certain conditions** so that I can **update my plans easily**.

---

## 4. Payments

**Story 10:**  
As a **guest**, I want to **make secure payments using Stripe, PayPal, or Paystack** so that I can **confirm my booking**.

**Story 11:**  
As a **host**, I want to **receive automatic payouts** after a successful stay so that I can **earn revenue from my property**.

---

## 5. Reviews & Ratings

**Story 12:**  
As a **guest**, I want to **leave a review and rating** after my stay so that I can **share my experience with others**.

**Story 13:**  
As a **host**, I want to **respond to guest reviews** so that I can **address feedback and maintain a good reputation**.

---

## 6. Notifications

**Story 14:**  
As a **user**, I want to **receive email or in-app notifications** about bookings, payments, or cancellations so that I can **stay updated on my activity**.

---

## 7. Admin Management

**Story 15:**  
As an **admin**, I want to **view analytics and reports** so that I can **monitor system performance and user activity**.
